在Movie Night项目中苦于没有记录用户信息与权限，考虑到某外卖项目中也缺失了一些Spring Security的技术栈，特意在原生项目中添加。

其间有些更新的配置，与网上资源不尽相同，特此记录。

pom.xml依赖引入#

为了在项目中通过Spring Security这一强大工具进行简单的安全访问设置，需要引入：

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
    <version>3.2.10</version>
</dependency>

注：本项目设置jdk17

安全类配置#

config包中添加安全类，本项目只进行了最基本的配置

Spring Security功能配置#

建立SecurityConfig类，创建各类相关 Bean，后期由SpringBoot实现依赖注入：

  1. 安全过滤链Bean–>Spring Security更新后只能通过Lambda表达式进行配置

    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
// 以下采用spring security 6.1+ 的配置方式
    http
        // 跨域访问设置（允许）
        .cors(cors -> cors.configurationSource(corsConfigurationSource()))
        // 禁用CSRF保护
        .csrf(AbstractHttpConfigurer::disable)
        .authorizeHttpRequests(auth-> auth
            // 不需要拦截的接口
            .requestMatchers("/user/movie/register").permitAll()
            .requestMatchers("/user/movie/login").permitAll()
            .requestMatchers("user/movie/hello").permitAll()
            .anyRequest().authenticated() // 其他请求需要认证
        )
        // 异常处理
        .exceptionHandling(exception -> exception
            .accessDeniedHandler(accessDeniedHandler()))
        // 项目采用session记录用户信息，故配置此项
        .sessionManagement(sessionManagementCustomizer ->           sessionManagementCustomizer
            .sessionCreationPolicy(SessionCreationPolicy.IF_REQUIRED))
            .logout(logout -> logout
                .logoutUrl("/user/movie/logout") // 设置登出URL
                .logoutSuccessUrl("/user/movie/login") // 登出成功后跳转的URL（本项目此部分实际由前端控制）
                .invalidateHttpSession(true) // 使会话失效
            )
        // 认证链，可设置jwt认证，此处仅采用用户名密码认证
        .addFilterBefore(sessionAuthFilter(), UsernamePasswordAuthenticationFilter.class)
        // 禁用表单登录
        .formLogin(AbstractHttpConfigurer::disable);

    return http.build();
}

    认证链具体信息可参见：SpringSecurity中对过滤器的配置：addFilterBefore

    1. 配置 session 过滤 Bean：
    public SessionAuthFilter sessionAuthFilter() {
    return new SessionAuthFilter();
}
    1. 跨域访问 Bean：
    public CorsConfigurationSource corsConfigurationSource() {
    CorsConfiguration configuration = new CorsConfiguration();
    // 前端访问地址（允许访问）
    configuration.setAllowedOrigins(List.of("http://localhost:8889"));
    configuration.setAllowedMethods(Arrays.asList("GET", "POST", "PUT", "DELETE", "OPTIONS"));
    configuration.setAllowedHeaders(List.of("*"));
    configuration.setAllowCredentials(true);
    configuration.setMaxAge(3600L);

    UrlBasedCorsConfigurationSource source = new UrlBasedCorsConfigurationSource();
    source.registerCorsConfiguration("/**", configuration);
    return source;
}
    1. 拒绝访问处理
    public AccessDeniedHandler accessDeniedHandler() {
    return (request, response, accessDeniedException) -> {
        // 返回 JSON 格式的错误响应
        response.setContentType("application/json;charset=UTF-8");
        response.setStatus(HttpStatus.FORBIDDEN.value());
        response.getWriter().write("{\"code\":403,\"msg\":\"无权限访问，请先登录\"}");
    };
}

    配合前端设置403状态下的弹窗信息、重定向页面等

Session认证类#

配合Spring Security创建的类

public class SessionAuthFilter extends OncePerRequestFilter {
    @Override
    protected void doFilterInternal(HttpServletRequest request, HttpServletResponse response, FilterChain filterChain) throws ServletException, IOException {
        HttpSession session = request.getSession(false);
        if(session!=null){
            // 用户信息实体类
            UserInfo userInfo= (UserInfo) session.getAttribute("userInfo");
            if(userInfo!=null){
                // 构建认证对象
                UsernamePasswordAuthenticationToken auth=
                        new UsernamePasswordAuthenticationToken(
                                userInfo,
                                null,
                                // 默认设置
                                Collections.singleton(new SimpleGrantedAuthority("ROLE_USER"))
                        );
                SecurityContextHolder.getContext().setAuthentication(auth);
            }
        }
        filterChain.doFilter(request, response);
    }
}

前端拦截配置#

注：笔者不会前端，以下内容都是遵循d老师的指导完成，可能有重复配置，但基本功能可以实现
适用Vue项目

总体构建#

在App.Vue中，进行总体设置：

<script>
export default {
  mounted() {
    // 监听未处理的 Promise 错误
    window.addEventListener('unhandledrejection', event => {
      if (event.reason.response && event.reason.response.status === 403) {
        this.$router.push({ name: 'Login' });
        event.preventDefault(); // 阻止默认行为
      }
    });
  }
}
</script>

响应拦截器设置#

在api.js中，创建axios实例：

const apiClient = axios.create({
    baseURL: API_URL,
    withCredentials: true
});

添加响应拦截器：

处理错误状态码，可弹出错误信息、重定向页面等

apiClient.interceptors.response.use(
    response => {
        // 正常响应直接返回
        return response;
    },
    error => {
        if (error.response) {
            switch (error.response.status) {
                case 403:
                    // 跳转到登录页面
                    alert('没有权限访问该资源，请登录后重试噢');
                    router.push({ name: 'Login' });
                    break;
                case 401:
                    // 显示未授权提示
                    alert('登录已过期，请重新登录');
                    router.push({ name: 'Login' });
                    break;
                default:
                    console.error('请求错误:', error);
            }
        }
        return Promise.reject(error);
    }
);

将所有函数的返回对象改为响应拦截器对象：

export const login = (contact, password) => {
    // 以 apiClient 替代原有的 api
    return apiClient.post(`${API_URL}/login`, { contact, password });
}

路由认证#

在index.js中设置需认证的路由：

router.beforeEach((to, from, next) => {
  // 检查需要认证的路由
  const protectedRoutes = [
    '/movie-detail',
    '/watch-together'
    // 以下省略
  ];

  // 检查用户登录信息（从 localStorage 判断）
  const isAuthenticated = localStorage.getItem('userInfo') !== null;

  if (protectedRoutes.includes(to.path) && !isAuthenticated) {
    // 保存目标路由，登录后重定向
    next({
      name: 'Login',
      query: { redirect: to.fullPath }
    });
  } else {
    next();
  }
});
