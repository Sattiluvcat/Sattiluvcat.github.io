融合ES的Movie Night小巧思

主要考虑通过ES做电影拉取数据的缓存，避免每次打开电影详情都要向豆瓣发信息 && 以ES进行模糊搜索（主要是简介内容）

以下主要对ES的配置和应用做一点介绍（实际配置时没什么问题，主要是做记录）

引入依赖#

经典步骤之在pom.xml中引入ES依赖：

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-data-elasticsearch</artifactId>
</dependency>
<dependency>
    <groupId>org.springframework.data</groupId>
    <artifactId>spring-data-elasticsearch</artifactId>
    <version>5.4.0</version>
</dependency>

注：ES依赖项的版本会影响构建查询的方法名、方法用法等，因此在抄别人的查询方法时需要检查版本对应关系

创建缓存类#

将需要查询的数据与索引存入ES中

缓存类构建如下：

@Data
@Document(indexName = "movie_metadata") // ES中的索引名称
public class MovieMetaDoc {
    @Id
    private String id;

    @Field(type = FieldType.Integer)
    private Integer movieId;

    // 需要引入分词器查询的字段
    @Field(type = FieldType.Text, analyzer = "ik_max_word", searchAnalyzer = "ik_max_word")
    private String summary;

    @Field(type = FieldType.Keyword)
    private String officialRating;

    @Field(type = FieldType.Keyword)
    private String coverUrl;

}

缓存操作服务类#

定义对缓存的基本操作（存储、查询等）

@Service
public class CacheServiceImpl implements CacheService {
    private static final String INDEX_NAME = "movie_metadata";
    // 引入ES操作类实例
    private final ElasticsearchOperations esOperations;

    public CacheServiceImpl(ElasticsearchOperations esOperations) {
        this.esOperations = esOperations;
    }

    // 缓存存入ES
    public void cacheMovieMeta(Integer movieId, String summary, String officialRating, String coverUrl){
        MovieMetaDoc doc = new MovieMetaDoc();
        doc.setMovieId(movieId);
        doc.setSummary(summary);
        doc.setCoverUrl(coverUrl);
        doc.setOfficialRating(officialRating);
        // 根据实例构建ES存储语句
        IndexQuery query=new IndexQueryBuilder()
                .withId(movieId.toString())
                .withObject(doc)
                .build();
        // 存入索引
        esOperations.index(query, IndexCoordinates.of(INDEX_NAME));
    }
    // 获取缓存信息
    public MovieMetaDoc getCachedMovieMeta(Integer movieId){
        return esOperations.get(
                movieId.toString(),
                MovieMetaDoc.class,
                IndexCoordinates.of(INDEX_NAME)
        );
    }
}
